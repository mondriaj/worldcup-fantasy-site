# Public Copy Style Guide v1

Status: active for Final Round public copy polish.

## Principles

- Use simple language for public UI.
- Keep caveats honest, short, and close to the action they qualify.
- Frame tools as planning help, not a guarantee.
- Do not imply official FIFA endorsement.
- Do not use betting language.
- Use "source-backed" only when the underlying source actually verifies the claim.
- Avoid debug phrases such as static data path, artifact internals, optimizer internals, or model-version language in visible UI.
- Explain Third Place risk clearly: it can add scoring opportunity, but lineups may rotate.
- Explain optionality carefully: earlier kickoff can help only if the user's rules allow substitutions.
- Keep final-squad and lineup caveats precise: official fantasy-pool data is used, but independently source-backed final squads and confirmed XIs are not verified here.

## Approved Phrases

- "Final Round fantasy setup"
- "Includes Final and Third Place"
- "Earlier kickoff can add flexibility if your rules allow substitutions."
- "Third Place can be higher scoring, but lineups may rotate."
- "Verify official FIFA locks, substitutions, and lineups before final decisions."
- "Official fantasy-pool data is used; independently source-backed final squads are not verified."
- "Planning help, not a guarantee."

## Avoid Phrases

- "lock" when it implies certainty rather than a user-selected player lock or game lock
- "guaranteed starter"
- "confirmed XI" unless source-backed
- "safe points"
- "bet"
- "sure win"
- "official FIFA model"
- "fairness verdict"
- "conspiracy"

## Review Rule

Copy polish may update wording, labels, and caveats only. It must not change recommendations, projections, score predictions, fixture authority, rankings, Team Builder objective logic, public optimizer behavior, or generated model artifacts.
