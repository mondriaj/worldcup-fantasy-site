Risk is a 0 to 100 score where lower is better. It combines injury/status risk, minutes risk, card risk, weekly volatility, and bad-week risk.

    How The Stats Are Made

    Open this to see which fields come from source data and which fields are calculated estimates.

    1. World Cup Player Base

    Source: data/players.json, data/teams.json, data/fixtures.json, and source notes in data/dataSources.md.

    Used for: player name, country, team, group, roster status, position, club, league, team quality, and group-stage fixture context.

    Important: some squads are still preliminary or tracker-based until final official World Cup squad releases are complete.

    2. Performance Matching

    Source: data/playerRecommendationInputs_v0.json and the club/national-team sources documented in data/dataSources.md.

    Used for: club minutes, starts, goals, assists, clean sheets, cards, selected attacking stats, national-team qualifier usage, caps signals, starter confidence, expected minutes, and data confidence.

    Important: weak-source and estimated-minute rows stay in the model, but they carry lower confidence and higher source risk.

    3. Finance Risk Model

    Source: data/playerFinanceMetrics_v0.json and data/fantasyFinanceModel_v0.md.

    Used for: expected return, volatility, downside deviation, VaR, CVaR, bad-week probability, Sharpe-style score, Sortino-style score, Omega-style score, and strategy tilts.

    Important: official World Cup fantasy prices are not imported yet, so price-adjusted return fields remain blank in the source model.

    Performance Stats

    These are estimated fantasy points made from available season stats and EURO-style scoring rules. They are not official tournament points.

    Appearance points: appearances x 1.

    60-minute points: estimated 60+ minute matches x 1. A 60+ minute match is estimated as min(appearances, max(starts, floor(minutes / 60))).

    Goal points: goalkeeper or defender goal = 6, midfielder goal = 5, forward goal = 4.

    Assist points: assists x 3.

    Clean sheet points: goalkeeper or defender clean sheet = 4, midfielder clean sheet = 1, forward clean sheet = 0.

    Recovery points: floor(recoveries / 3).

    Goalkeeper save points: for goalkeepers only, floor(saves / 3).

    Penalty save points: penalties saved x 5.

    Goals conceded penalty: for goalkeepers and defenders, floor(goals_conceded / 2) x -1.

    Negative points: yellow cards x -1, red cards x -3, own goals x -2, missed penalties x -2.

    Total estimate: all the positive points and negative penalties above added together.

    Example: Appearance points

    If a player appeared in 24 matches, appearance points = 24 x 1 = 24.

    Example: 60-minute points

    If a player made 18 appearances, 12 starts, and played 1,050 minutes, then estimated 60-minute matches = min(18, max(12, floor(1050 / 60))) = min(18, 17) = 17. 60-minute points = 17.

    Example: Goal points

    If a midfielder scores 7 goals, goal points = 7 x 5 = 35. If a forward scores 7 goals, goal points = 7 x 4 = 28.

    Example: Assist points

    If a player has 9 assists, assist points = 9 x 3 = 27.

    Example: Clean sheet points

    If a defender has 10 clean sheets, clean sheet points = 10 x 4 = 40. If a midfielder has 10 clean sheets, clean sheet points = 10 x 1 = 10.

    Example: Recovery points

    If a player has 137 recoveries, recovery points = floor(137 / 3) = 45. The extra 2 recoveries do not make another full point.

    Example: Goalkeeper save points

    If a goalkeeper has 88 saves, goalkeeper save points = floor(88 / 3) = 29. This only applies to goalkeepers.

    Example: Penalty save points

    If a goalkeeper saves 2 penalties, penalty save points = 2 x 5 = 10.

    Example: Goals conceded penalty

    If a defender's team conceded 31 goals, goals conceded penalty = floor(31 / 2) x -1 = -15. This penalty applies only to goalkeepers and defenders.

    Example: Negative points

    If a player has 5 yellow cards, 1 red card, and 1 missed penalty, negative points = 5 x -1 + 1 x -3 + 1 x -2 = -10.

    Example: Performance total

    A midfielder with 20 appearances, 15 estimated 60-minute matches, 6 goals, 5 assists, 8 clean sheets, 90 recoveries, and 4 yellow cards gets: appearances 20, minutes 15, goals 6 x 5 = 30, assists 5 x 3 = 15, clean sheets 8 x 1 = 8, recoveries floor(90 / 3) = 30, cards 4 x -1 = -4. Total estimate = 114.

    Small-Sample Correction: Confidence Adjustment

    This protects the builder from overrating a player who played very little but had one big moment. Instead of trusting every raw per-90 number equally, we blend the player's raw rate with the average rate for their position.

    Raw points per 90: total performance estimate divided by 90s played.

    Position baseline: average raw points per 90 for players in the same position.

    Confidence weight: 90s / (90s + 8). A player with more minutes gets a weight closer to 1. A player with very few minutes gets pulled closer to the position average.

    Sample confidence score: confidence weight x 100. This is the 0 to 100 version used inside Reliability.

    Confidence-adjusted per 90: confidence_weight x player_raw_per90 + (1 - confidence_weight) x position_average_per90.

    Confidence-adjusted expected points: confidence-adjusted per 90 multiplied by the player's average minutes per appearance divided by 90.

    Example: Raw points per 90 with 60 minutes played

    If a player has a total performance estimate of 6 points from 60 minutes, then 90s played = 60 / 90 = 0.67. Raw points per 90 = 6 / 0.67 = 9.0. This looks excellent, but it is based on a tiny sample.

    Example: Position baseline

    If defenders in the database have raw points per 90 of [3.8, 4.2, 5.0, 5.4], the defender baseline = (3.8 + 4.2 + 5.0 + 5.4) / 4 = 4.6.

    Example: Confidence weight

    If a player has played 0.67 full-match equivalents, confidence weight = 0.67 / (0.67 + 8) = 0.08. The model trusts only 8% of the player's raw rate and leans heavily on the position baseline.

    Example: Sample confidence score

    If confidence weight = 0.08, sample confidence score = 0.08 x 100 = 8. If confidence weight = 0.79, sample confidence score = 79.

    Example: Confidence-adjusted per 90

    Using raw per 90 = 9.0, position baseline = 4.6, and confidence weight = 0.08, confidence-adjusted per 90 = 0.08 x 9.0 + 0.92 x 4.6 = 4.95.

    Example: Confidence-adjusted expected points

    If confidence-adjusted per 90 = 4.95 and the player usually plays 60 minutes, expected points per appearance = 4.95 x (60 / 90) = 3.3.

    Example: why the correction matters

    Player A has a raw score of 10 points per 90, but only played 2 full-match equivalents. The position average is 5. Confidence weight = 2 / (2 + 8) = 0.20. Confidence-adjusted per 90 = 0.20 x 10 + 0.80 x 5 = 6.0. The player still gets credit, but not full credit for a tiny sample.

    Example: a larger sample gets trusted more

    Player B also has a raw score of 10 points per 90, but played 30 full-match equivalents. Confidence weight = 30 / (30 + 8) = 0.79. Confidence-adjusted per 90 = 0.79 x 10 + 0.21 x 5 = 8.95. Because the sample is much larger, the estimate stays closer to the player's own rate.

    Risk Measures

    Risk fields use weekly fantasy point history, player status, chance of playing, minutes, starts, appearances, cards, and the player's bad-week pattern. Missed penalties are counted in performance scoring, but not in discipline risk.

    Volatility: standard deviation of weekly fantasy points. It measures how much the player moves around their own average.

    Downside deviation: downside-only movement below a 2-point target. Note: this is not directly comparable to volatility because good weeks are ignored and only bad shortfalls count.

    Bad-week rate: share of weeks with 1 point or fewer.

    Value at Risk 10: rough 10th percentile weekly score.

    Conditional Value at Risk 20: average of the worst 20% of weekly scores.

    Availability risk: an index from 0 to 100. It uses status penalty + (100 - chance of playing) x 0.5 + missed-game share x 35.

    Minutes risk: start rate, 60-minute rate, and average minutes per appearance.

    Discipline risk: yellow cards per 90 and red cards per 90. Penalties committed would fit here, but the current source data does not include them.

    Tail risk: a weighted bad-week index: bad-week rate has weight 55, Value at Risk shortfall has weight 12, and Conditional Value at Risk shortfall has weight 10.

    Composite risk: availability 28%, minutes 24%, discipline 12%, volatility 20%, tail risk 16%.

    Example: volatility

    If a player scores [2, 3, 4, 5], their weeks are steady. If another player scores [0, 0, 8, 10], their average may look useful, but the standard deviation is much higher.

    Example: downside deviation

    Use a target of 2 points. For weekly points [8, 2, 1, 0], the downside shortfalls are [0, 0, 1, 2]. Downside deviation is sqrt((0^2 + 0^2 + 1^2 + 2^2) / 4) = 1.12. This should not be compared one-to-one with volatility because the 8-point good week counts as zero downside.

    Example: bad-week rate

    For weekly points [0, 1, 2, 4, 8], bad-week rate is 2 / 5 = 40% because two weeks are 1 point or lower.

    Example: Value at Risk 10

    For sorted weekly points [0, 1, 2, 4, 8], the 10th percentile is close to the low end of the distribution. It answers: "What does a pretty bad week look like for this player?"

    Example: Conditional Value at Risk 20

    For weekly points [0, 1, 2, 4, 8], the worst 20% is the single worst week, so CVaR 20 = 0. With 10 weeks, it would average the worst 2 weeks.

    Example: availability risk

    Status penalty is based on the source status field: available = 0, doubtful = 25, injured/suspended = 40, unavailable = 45, and not available = 35. If a player has a doubtful status penalty of 25, a 75% chance of playing, and missed 20% of gameweeks, availability risk = 25 + (100 - 75) x 0.5 + 0.20 x 35 = 44.5, rounded to 45.

    Example: minutes risk

    If start rate = 70%, 60-minute rate = 60%, and average minutes = 65, minutes risk = (1 - 0.70) x 45 + (1 - 0.60) x 30 + (75 - 65) x 0.7 = 32.5, rounded to 33.

    Example: discipline risk

    If yellow cards per 90 = 0.25 and red cards per 90 = 0.02, discipline risk = 0.25 x 14 + 0.02 x 45 = 4.4, rounded to 4. Missed penalties are not included here because they are finishing events, not discipline events.

    Example: tail risk

    If bad-week rate = 30%, Value at Risk 10 = 0, and CVaR 20 = 0.5, tail risk = 0.30 x 55 + (2 - 0) x 12 + (2 - 0.5) x 10 = 55.5, rounded to 56. This intentionally gives the most weight to how often a player has very bad weeks.

    Example: composite risk

    If availability risk = 20, minutes risk = 30, discipline risk = 10, volatility risk = 40, and tail risk = 25, then composite risk = 20 x 0.28 + 30 x 0.24 + 10 x 0.12 + 40 x 0.20 + 25 x 0.16 = 26.0.

    Risk-Adjusted Scores Used By The Builder

    Reliability score: sample confidence score 45%, low availability risk 30%, and low minutes risk 25%.

    Risk-adjusted expected points: confidence-adjusted expected points reduced when composite risk is high: points x (1 - risk / 150).

    Risk-Adjusted Pick (Sharpe-style): balances projected points against overall risk. Raw ratio = (expected points - 2) / weekly point volatility, then converted to a 0 to 100 index by percentile rank in the player database.

    Downside Protection Pick (Sortino-style): focuses more on avoiding bad outcomes. Raw ratio = (expected points - 2) / downside deviation, then converted to a 0 to 100 index by percentile rank in the player database.

    Best Overall score: combines risk-adjusted expected points, reliability, and lower composite risk into a 0 to 100 score.

    Example: reliability

    If sample confidence score = 80, availability risk = 20, and minutes risk = 30, reliability = 80 x 0.45 + (100 - 20) x 0.30 + (100 - 30) x 0.25 = 77.5.

    Example: risk-adjusted expected points

    If confidence-adjusted expected points = 5.0 and composite risk = 30, risk-adjusted expected points = 5.0 x (1 - 30 / 150) = 4.0.

    Example: Risk-Adjusted Pick (Sharpe-style)

    If expected points = 5, baseline = 2, and weekly volatility = 3, raw Sharpe-style ratio = (5 - 2) / 3 = 1.0. If that raw ratio ranks around the 75th percentile among players, the displayed Risk-Adjusted Pick index is about 75.

    Example: Downside Protection Pick (Sortino-style)

    If expected points = 5, baseline = 2, and downside deviation = 1.5, raw Sortino-style ratio = (5 - 2) / 1.5 = 2.0. If that raw ratio ranks around the 85th percentile among players, the displayed Downside Protection Pick index is about 85.

    Example: Best Overall score

    Best Overall gives most weight to risk-adjusted expected points, then adds reliability, then rewards lower composite risk. This is the default team-builder score because it avoids choosing only upside players or only low-risk players.

    Stats Shown On This Page

    These labels appear in the dashboard, captain table, team advice table, lock-player list, and team builder cards. They are listed here at the end because they combine the source fields and calculated fields above.

    Price: official World Cup fantasy prices are pending. The current builder shows proxy_price_v1, a prototype placeholder used only to test budget and value logic.

    Captain Score: a captain shortlist score using expected return, risk-adjusted return, reliability, composite risk, and tail risk.

    Reliability: a 0 to 100 minutes-security score from club minutes, starts, national-team usage, source confidence, and availability-style risk.

    Start %: prototype probability that the player starts for his country in the selected matchday view.

    Expected Minutes: prototype minutes estimate using national-team usage first, then club usage, roster status, captain signal, and country position depth.

    Substitution Risk: a 0 to 100 risk score for limited minutes, bench role, or early substitution. Lower is better.

    Per 90: modeled fantasy upside if the player played a full 90 minutes.

    Risk: a composite score from source risk, minutes uncertainty, volatility, downside risk, fixture risk, discipline, and bad-week probability. Lower is better.

    Style Score: the score for the selected pick style, such as Best Overall, Projected Points, Reliable Pick, Upside Pick, Likely Minutes, Best Value Prototype, Cheap Enabler, Premium Worth It, Sharpe-style, Sortino-style, VaR, CVaR, Omega-style, Attack Heavy, Defensive Heavy, or Very Risky Upside.

    Projected: risk-adjusted expected fantasy points for one group-stage match.

    VaR And CVaR: downside floor measures. VaR is the modeled 10th percentile outcome, and CVaR is the modeled average of the worst 20% of outcomes.

    Attack Heavy: tilts toward attacking players and attacking defenders with strong goal, assist, shot, and upside signals.

    Very Risky Upside: intentionally searches for boom-or-bust players. It is useful for aggressive watchlists, not safe default picks.

    Example: Price

    Until official prices are imported, each player uses proxy_price_v1. It is built from position, role-adjusted expected return, starter confidence, expected minutes, team quality, and risk. It is not an official price.

    Example: Captain Score

    If a player has expected return of 5.4, risk-adjusted return of 4.8, reliability of 82, risk of 30, and tail risk of 20, the captain score rewards the returns and reliability, then subtracts risk penalties.

    Example: Reliability on the page

    A reliability score of 85 means the player has strong data confidence, looks available, and usually plays meaningful minutes. A score of 35 means the player may be useful but has more playing-time uncertainty.

    Example: Per 90 on the page

    If the card says Per 90: 6.2, it means the player's estimated performance is 6.2 fantasy-style points for a full 90 minutes before adjusting for confidence and typical minutes.

    Example: Risk on the page

    If Risk = 22, the player is relatively safer in the loaded dataset. If Risk is near the top of the loaded dataset, check availability, minutes, discipline, volatility, and bad-week concerns before relying on that player.

    Example: Style Score

    If Pick Style is Reliable Pick, the score shown in the lock list ranks players by low composite risk plus useful projected points. If Pick Style changes to Upside Pick, the score changes to focus on points per 90.

    Example: Best Overall style

    Best Overall uses the 0 to 100 balanced score. A player with strong projected points, high reliability, and low risk should beat a player who is excellent in only one category.

    Example: Projected Points style

    If Player A has risk-adjusted expected points of 4.8 and Player B has 3.9, Projected Points style ranks Player A higher, even if Player B is cheaper.

    Example: Reliable Pick style

    If composite risk = 25 and expected points = 4, Reliable Pick score = (100 - 25) + 4 x 8 = 107. It rewards low risk first, then adds expected points.

    Example: Upside Pick style

    If a player has Per 90: 7.4, Upside Pick likes that player because they produce a lot when they are on the field. The tradeoff is that this style may select players with less certain minutes.

    Example: Likely Minutes style

    If Player A has reliability 88 and Player B has reliability 52, Likely Minutes style ranks Player A higher because the goal is to avoid players who might not start or might play very little.

    Example: Avoid Bad Weeks (low tail-risk)

    If tail risk is low, VaR is positive, and CVaR is not painful, Avoid Bad Weeks likes the player even if another player has flashier upside.

    Example: Risk-Adjusted Pick (Sharpe-style) on the page

    If Risk-Adjusted Pick = 82, that player ranks around the 82nd percentile for reward above baseline per unit of weekly volatility in this database.

    Example: Downside Protection Pick (Sortino-style) on the page

    If Downside Protection Pick = 88, that player ranks around the 88th percentile for reward above baseline per unit of downside-only volatility. This is useful when you care more about bad surprises than all volatility.

    Example: Projected

    If expected return is 5.0 and the risk-adjusted return is 4.0, the page shows the risk-adjusted number as Projected because it is closer to the practical fantasy recommendation.

    Example: VaR and CVaR

    If VaR is 2.0, the modeled bad-outcome cutoff is still positive. If CVaR is -3.0, the worst-outcome basket is much weaker, even if the headline projection looks useful.

    Example: Very Risky Upside

    A player can rank well here because the model sees high upside and volatility together. That makes the player interesting for an aggressive portfolio, but not automatically a safe starter.

    Important limits: These are learning-project estimates, not official World Cup predictions. Some official fantasy stats are missing because the source data does not include them yet, including Player of the Match, penalty won, penalty conceded, and goals from outside the box.
